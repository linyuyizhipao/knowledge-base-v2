```
  ┌─────────────┐              ┌─────────────┐

  └─────────────┘              └─────────────┘
```

## 项目职责分工

| 项目 | 职责 | 核心模块 |

## 核心流程

### 1. 支付创建流程 (pay/create)

```
用户发起支付请求
       │
       ▼
┌─────────────────────────────┐
│ slp-php: pay/create         │
│ 1. 判断支付类型              │
│    - 充值/打赏/商品/贵族等   │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 2. 非充值类型：钻石足够？   │
│    YES → 扣钻石逻辑，return  │
│    NO  → 继续下方流程        │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 3. 写入 xs_pay 表            │
│    - type: 业务类型标识      │
│    - 支付金额/订单信息        │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 4. 调用 slp-ee-config       │
│    创建支付订单              │
└─────────────┬───────────────┘
              │
              ▼
       跳转微信支付
```

### 2. 支付回调流程

```
用户完成支付
       │
       ▼
┌─────────────────────────────┐
│ 微信支付回调                 │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ slp-ee-config: PayCommon.php│
│ 1. 更新 xs_pay 记录           │
│ 2. 发布 NSQ: xs.pay          │
└─────────────┬───────────────┘
              │
              ▼
       NSQ Topic: xs.pay
              │
              ▼
┌─────────────────────────────┐
│ slp-php: PayTask.php        │
│ 消费者：xs.pay 进程           │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 根据 xs_pay.type 执行业务逻辑 │
│ - packet: 房间打赏           │
│ - recharge: 充值             │
│ - goods: 商品购买            │
│ - vip: 贵族购买              │
└─────────────────────────────┘
```

## 数据表结构

### xs_pay

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint64 | 主键 |
| `uid` | uint32 | 用户 ID |
| `type` | string | 业务类型（packet/recharge/goods/vip 等） |
| `amount` | uint32 | 支付金额（分） |
| `order_id` | string | 支付订单号 |
| `status` | uint32 | 支付状态 |
| `create_time` | uint32 | 创建时间 |
| `pay_time` | uint32 | 支付时间 |
| `extra` | text | 扩展数据（JSON） |

---

## NSQ 事件

### xs.pay

| 属性 | 说明 |
|------|------|
| **Topic** | `xs.pay` |
| **发布方** | slp-ee-config |
| **消费方** | slp-php (PayTask.php) |
| **触发时机** | 支付回调成功，xs_pay 更新完成后 |
| **事件数据** | xs_pay 记录 ID |

---

## 业务类型 (type)

| type | 说明 | 业务逻辑 |
|------|------|----------|
| `packet` | 房间打赏 | 根据支付 ID 找到 xs_pay，继续执行打赏逻辑 |
| `recharge` | 充值 | 增加用户余额/钻石 |
| `goods` | 商品购买 | 发放商品 |
| `vip` | 贵族购买 | 开通贵族身份 |
| `...` | 其他 | 根据业务扩展 |

## 核心代码位置

### slp-php

| 文件 | 说明 |
|------|------|
| `pay/create` | 支付创建入口（统一处理充值、打赏、商品、贵族等） |
| `cli/tasks/PayTask.php` | xs.pay NSQ 消费者，根据 type 执行业务逻辑 |

### slp-ee-config

| 文件 | 说明 |
|------|------|
| `pay/PayCommon.php` | 支付回调处理，更新 xs_pay 并发布 NSQ |

## 问题排查

### 问题 1: 支付成功但未执行业务逻辑

**排查步骤**：
1. 检查 xs_pay 表中 `type` 字段是否有值
2. 检查 slp-ee-config 日志，确认 NSQ 是否发布
3. 检查 slp-php 的 PayTask.php 是否在消费 xs.pay
4. 查看 PayTask.php 日志，确认 type 路由是否正确

### 问题 2: 非充值类型扣费异常

**排查步骤**：
1. 检查 pay/create 中钻石余额判断逻辑
2. 确认 xs_pay 写入是否成功
3. 检查 slp-ee-config 订单创建是否成功

### 问题 3: 小程序支付配置错误

**排查步骤**：
1. 核对 appid 与 user_name 是否匹配
2. 检查商户 ID 和商户 KEY 配置
3. 确认小程序主体权限
